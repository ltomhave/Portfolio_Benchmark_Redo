Wells Fargo Web App

UI|UX Designer for Web Application

• THE PROJECT:
To partner within an agile team of engineers in the Wells Fargo organization to design and develop the user interface for an internal real estate evaluation system to be used by appraisers to collect data and conduct appraisals.

• THE GOALS:
To create a web application that would streamline, simplify, and standardize the appraisal documentation process for real estate appraisers.

• THE CHALLENGES:
— No exposure to actual users as management felt that the above assets were enough for the team
— The existing excel document the appraisers used was complex and dynamic. It was hard to determine each field and use case based on just working through the document and training videos
— Extreme time constraints. Management wanted the application launched and able to use within 6 months

• THE OUTCOMES:
— A robust prototype of the commercial appraisal web application that streamlined the user experience by reducing the input of duplicative information and provided easy ways to access, edit, and attach images, maps, and other addendum's to complete the appraisal.
— HTML/CSS templates of the main form pages to ensure the responsive design, interaction, and visual system are the same across the application.

• TOOLS USED:
Excel Figma Illustrator HTML CSS Github



  • View Figma Prototype
  • View Style Guide

See the case study below

1  -  Creating a Plan

The planning stage consisted of 3 parts. Gathering and understanding the application's requirements by learning the current commercial appraisal process and tools, analyzing what we can use of the existing residential web application that the commercial app would be based off of, and meeting with the product owner and lead engineers to formulate a working road map.

This was both the easiest and hardest part of the project. The appraisal team already had a robust and dynamic excel document for creating appraisals as well as good training videos of how to use it. There was also an existing application for the residential appraisal team that was seen as a good starting point for the commercial application. This made it easy to walk in the user's footsteps and gain empathy for their process and pain points. The downside was because we had those tools, management did not feel we needed to take up the user's time by working with them directly, despite pushing for direct user input. As we worked to streamline the process, this did lead to some educated speculation on behalf of the user. In hindsight, it would have been ideal to go through the high fidelity prototype with the users and get sign off prior to proceeding with development. I was only involved in the user interface design but anticipate this causing more rework for the engineer team.

Below are examples of the original excel document that we were given against the web application pages. While robust, the document was also very complex, consisting of at least 50 different pages and multiple dynamic text fields.

2  -  Creating a Framework

Once I understood the requirements and current state of the commercial appraisal business, I was able to compare against the existing residential app to determine what we could use, what could be improved, and what needed to be done from scratch from a user experience perspective. This was in partnership with the lead engineers who did the same thing from their perspective. In reality, only about 50% of the existing framework from residential was usable. The residential application form UI was a PDF format that wasn't easily scannable for the user. It also wasn't responsive, which made it very hard to view on different devices.

Our combined analysis allowed us to partner on creating a general framework for the application. Text fields were cataloged with duplicative information reduced or combined in a separate excel doc by page. I then created a quick medium fidelity black/white mockup of the first few pages as a jumping off point to discuss the visual framework and conventions with internal team and get early buy in.

3  -  Designing

I began the UI piece by analyzing the Wells Fargo existing customer facing design system and referring to it as a base. As we progressed I modified the color and visual design system based on the the specific business needs. For example, I chose levels of the standard yellow to represent various hierarchies within the form. I also reduced the amount of space between form fields to help reduce the total length of the form while still maintaining readability. Due to time constraints and the need to multi task this process happened in tandem with creating the mid and high fidelity prototypes. I documented all the visual standards in our design system document for referral.

The prototype was created and signed off on in 5-6 page groupings, again due to time restraints and the need to pass pages off to the engineering team during the next spring. This approach had the advantage of allowing us to quickly move to development but also created a lot of rework as we were always re-analyzing the complex and dynamic nature of the application and working through total user flows in pieces. Part of my role was to bridge the gap between this process and an engineering team that was spread across 2 countries and time zones, relaying any changes, style updates, etc. To ensure clarity I took a multi-pronged approach to communication. During daily stand ups I would walk the team through any new information and reiterate changes via e-mail, which acted as a running change log. I also ensured the prototype, CSS, and design system document were up to date and the team had access to the files.

Once the mockup was signed off on by the product manager, I coded the initial HTML and CSS in Visual Code Studio for each type of page as a template to pass off the software engineers. Below is an example of the responsive design across 4 different devices.

4  -  Testing

Due to time constraints and lack of access to the end user, the initial testing I was involved in was done internally with our product group. As each section was completed I partnered with engineer team to walk through and test application clarify, functionality, accessibility, and general usage. I also ensured the visual system and interactions met the quality and visual standards across the application.

Given different time restraints, Iâ€™d want to be involved in troubleshooting and gaining feedback upon the release to the business users. Ideally, this would be with a core group of business users that could provide structured feedback that would allow us to improve the application before a wide release.

5  -  Hindsight and Future Steps

My biggest takeaway from this project was the ability to work and solve problems independently as well as part of a team. As the requirements for the role changed I was tasked with coding templates in HTML and CSS, a weaker skill set before this role. The reality of the tight time frame meant that no one else on the team had the time to hold my hand as I learned. I was able to be successful in this role by taking responsibility for finding online resources and figuring it out as I went, a true trial and error process.

In reevaluating the design, I would have liked to have more time during the planning stage to work through information architecture. For instance, there were a lot of menus with different functionality and I think we could have spent more time breaking down the actual information architecture to make it more clear for the user and relied less on the existing excel documents and systems.

Belk

Customer Experience Designer

• THE ROLE:
To leverage well crafted strategies to delight users at each step of the conversion journey and nurture strong customer-brand relationships with the Crown & Ivy Kid's brand for Belk.

• THE CHALLENGES:
– Establishing and sustaining cross functional partnerships across multiple channels to ensure the brand quality and message was consistent at each touch point.
– Connecting with a very different demographic in order to gain empathy and fighting for user-centric solutions for their pain points and goals.
– Creating a brand identity that stands out while fitting into a larger department store organization of national and private brands.

• THE GOALS:
To create a compelling and positive brand experience that would draw the customer in and keep them coming back for more.

• THE RESULTS:
The largest and most popular private kid's brand within Belk which resulted in $40M per year in revenue.

• TOOLS USED:
Excel Adobe PLM

After a year of success designing for the Missy brand, I was given the opportunity to launch and manage the customer experience of this kid's line.

New to childrenswear, my first step was to conduct extensive research on the market and to gather any existing information available within the Belk organization on the target demographic. I partnered with merchant teams to get intel on the other kid's brands. This helped me learn what generally works, what doesn't, and further identify the product and aesthetic gap. I then used my market research to identify benchmark brands within the kid's sector that aligned with our target customer and the missy aesthetic. Feathered Friends

UI|UX Website Design

• THE PROJECT:
The Feathered Friends bird rescue wants to update and overhaul their website to streamline the experience for a user to view, support, foster, and adopt birds.

• THE CHALLENGES:
The current website is older and hard to navigate and there isn't much information about the birds available. The secondary challenge is that with a small volunteer staff the rescue is overwhelmed with the current workload of paper forms.

• THE GOALS:
To increase bird adoption and retention rate by 15% by connecting the right people with the right bird.

• TOOLS USED:
Excel Figma Photoshop



  • View Site Demo
  • View Style Guide

See the case study below

1  -  Kicking It Off

I started my research with a deep dive questionnaire exercise with the business stakeholders (in this case my design mentor)to get qualitative research on the goals and frustrations of both the user and rescue operation. I then conducted extensive market research of other animal and bird specific adoption and rescue sites and apps in order to quantify comparable services and digital features, distilling this information into a user and business profile.

2  -  Creating a Framework

As this was a hypothetical business I created for this project, part of the challenge was defining the brand and current state of the rescue as well as designing the solutions. Based on my research of real world rescues and in partnership with my design mentor, I envisioned my rescue as having a very outdated website that was hard to navigate and cumbersome to actually get or provide information on. I was also able to use my previous branding experience to flesh out the visuals such as a logo and color scheme for this brand.
At this point, I was able to define the problem statement and create the below creative brief to help focus and guide my efforts. I did this by reviewing the data I had collected along with the branding I had created and pulled out the key points and takeaways that I wanted to focus on.

In order to further identify pathways and opportunities, I created a user journey flow using my "current state" information. At this step I was able to identify a few new pain-points opportunities to improve in my design. These included: anxiety around whether or not a form was sent/received and a desire for the opportunity to share the bird with friends or take more time to think about the bird before adopting.
I then worked up a user map and site map to help map out my "future state". I included solutions to some of the pain points above by including confirmation messages when a form is sent and a favorites section the user can save birds in to view later as well as the opportunity to share the bird to social media. Below you can see the pathway for a user looking to adopt a parrot, which includes the above.

3  -  Designing

I choose to focus on the full screen desktop design for the prototype as initial research indicated that users were more likely to fill out complex forms on a desktop than mobile devise. I started with wire frames and did some preliminary A/B user testing on content layout. This testing showed that each page was too high level and it took too many steps to get to the desired information. In the redesign I focused on making each page more information rich while still being easily scannable for quick information.



Once I had worked through those bugs, I was able to combine the existing research and my own background in visual design to create compelling high-fidelity prototypes reflective of the final web site. Next steps would include creating a guideline for the responsive design of each section of the website.

4  -  Testing

In order to prove that my design solved the challenges and achieved the business goals identified during kickoff, I created and performed a qualitative usability test. I decided to test the 2 tasks outlined below based on the priority of importance to meeting the objectives as well as the complexity of the tasks. The below outlines my testing plan and objectives as well as the moderator speech I gave prior to conducting the test.

The below notes summarize the test results and takeaways from each task. The overarching takeaways were to clarify any ambiguous naming conventions, reassure the user as to the confidentiality of the application, and ensure the user is able to quickly select the type of class or volunteer opportunity they are looking for when signing up.

The solutions to the above testing challenges are included in final prototype with a quick before and after overview provided below.

5  -  Hindsight and Future Steps

I believe that I needed to spend more time during the planning stage on information architecture and naming conventions. I would do this by conducting a tree test to help clarify naming conventions and flesh out the card sort with additional levels of complexity.

Going forward, I would expand usability testing to a more diverse audience to see out any unaddressed pain points or accessibility concerns. I would also conduct market research on the need and appetite for a mobile app.

Resilient Cities & Communities

Branding and Product Design

• THE PROJECT:
To create branding materials inclusive of font, color, and digital assets for the launch of a new non-profit.

• THE CHALLENGE:
As a new organization, the primary challenge was to understand the brand vision and create style guidelines from scratch. There was an additional challenge of partnering on a creative project with a person without a design background.

• THE GOALS:
To develop branding guidelines and delivery high quality digital products for website usage.

• A TESTIMONIAL:
"Lara is fabulous to work with providing professional, quick, and quality designs. Within a couple of weeks, we received a full branding identity, graphics, icons, and logos. Absolutely exceptional quality and professional delivery in split-second flash timing. So very pleased and grateful! Thank you Catchafire and Lara! Our startup nonprofit is totally impressed and blessed!"
Vicki O'Day, Innovation and Advancement Director

• TOOLS USED:
Excel Illustrator Photoshop PowerPoint



  • Website Link
  • View Style Guide

See the case study below

1  -  KICKING IT OFF

I first guided the nonprofit representative through the creative process and ensured mutual clarity on project, goals, end user, and brand vision through a qualitative interview. Once I analyzed the feedback from this interview and the market and user research I conducted on my end, I decided to use a colorful palette to reflect of the brands vision of multi-cultural inclusively. First, I guided the non-profit representative through the creative process and clarified expectations, deliverables, and timeline. I then partnered with the representative to analyze the data they had available on the target customer and organizational objectives in order to provide targeted visual branding identity and digital assets. I next spent time researching and analyzing the brand vision of other non-profits with similar end goals to understand the market.

2  -  FORMULATING A PLAN

After analyzing the data, we started to further define the user and what brand voice we wanted to use to reach them. We identified the user as a 30-50 year old mid-to-high income professionals from a variety of different cultures who live in the Minneapolis area and want to take action in their communities to make them more resilient to climate change. The voice was strong and inspiring.

We then aligned on a project plan that was inclusive of all the required design and branding deliverables and TNA (shown below). These included an updated style guide, a logo, and custom logo and digital assets.

3  -  CREATING THE PRODUCT

With an understanding of the user and focused directive, I then began developing potential solutions to the design challenges. I started by creating a mood and color board inclusive of typeface in order to get early buy in from the organization representative on the overarching visual direction. We had some great dialogue around the mood board and after some discussion decided to add the multi-colored interlocked hands imagery that reflected the goal of engaging diverse cultures.

With a narrowed direction I could start to create and iterate the logo and digital assets. Using adobe photoshop and illustrator, I worked through different color-ways and styles of design deliverables until I was satisfied with 3-4 to present to the client.

For the logo, I chose a tree to represent the climate aspect and then partnered it with a person as the trunk to symbolize the union of man and nature.

The representative had done some great website pre-planning so we were able to identify up front the specific areas we wanted to symbolize with icons and graphics. I first presented generic market images I thought visually represented each area well in order to get feedback on the icon visual direction and then proceeded to sketch and iterate each one in the same hand and palette as the logo.

4  -  WRAPPING UP

As this was a small non-profit, the testing phase was confined to the client and board leadership. This stage was more of a collaborative back and forth A/B testing as we discussed the visuals and any challenges or alterations that needed to be addressed in order to ensure the assets provided optimal design solutions.

Given additional resources or the opportunity to be involved in the organization post launch, I would very much like to conduct additional research on the target user vs. the actual user in order to tweak any colors, visuals, or tones in the branding. I'd also like to get metrics to analyze around the website and conduct focus groups of the actual users in order to get feedback on how people view the site and the overarching impression they get from the visuals.

5  -  HINDSIGHT

•What I did well:
-The planning and communication was robust and extremely productive. This was achieved by doing extensive up front research and encouraging a reciprocal working relationship with the business representative that resulted in a shared understanding of the client's needs, expectations, and deliverables.
-The handoff of digital assets was well organized and easy for the client to work with. This was achieved by utilizing a shared google drive that contained the digital assets as well as an intuitive style guide that recorded the version history of the assets.

•What could be improved:
-There was an opportunity on my part to improve communication by being more selective and brief when presenting my deliverables. Instead of presenting the number of options I should have had a more specific point of view with which to guide the business representative.
